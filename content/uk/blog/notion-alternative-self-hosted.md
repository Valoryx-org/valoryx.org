Notion скрізь. Інженерні команди дедалі частіше використовують його для технічної документації. Це працює — до певної межі.

## Що Notion робить правильно
Швидкий гнучкий редактор. Корисні представлення баз даних. Підходить для нетехнічної документації.

## Де Notion дає збій

**Немає інтеграції з git.** Документи у власній базі даних Notion, відірвані від вашого коду. Розбіжність документації практично неможливо уникнути.

**Прив'язка до платформи.** Експорт у markdown є неохайним. Ваша документація перетворюється на заручника.

**Лише хмарний варіант.** Немає self-hosted Notion. GDPR, резидентність даних — заблоковано.

**Продуктивність при масштабуванні.** Великі воркспейси гальмують. Поганий повнотекстовий пошук.

**Ціна при масштабуванні.** $8 на користувача на місяць. Команда з 20 осіб = $1,920 на рік лише за документацію.

## На що звертати увагу
1. Справжня двостороння інтеграція з git
2. Self-hosting
3. Вихід у форматі markdown
4. Вебредактор
5. Публікація документів
6. Фіксована ціна

## Порівняння варіантів

**Valoryx:** git І Є сховищем. WYSIWYG-редактор. Один бінарний файл. 5 редакторів безкоштовно. Публікація документів вбудована.
**Wiki.js:** Self-hosted, база даних на першому місці, git — опціонально.
**Docusaurus:** Суто git, без вебредактора.
**Confluence:** Корпоративний рівень, Atlassian, без git.
**Outline:** Self-hosted, чистий редактор, без git.

## Прихована вартість Notion
Розбіжність документації. Складність міграції. Переключення контексту. Труднощі з пошуком.

```bash
curl -fsSL https://valoryx.org/install.sh | sh
docplatform init --workspace-name "My Docs" --slug my-docs
docplatform serve
```
