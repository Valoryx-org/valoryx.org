Самостійно розгорнута документація переживає справжній підйом. Після того як роки поспіль SaaS-платформи підвищували ціни, змінювали умови та без попередження припиняли роботу, дедалі більше команд обирають власну інфраструктуру документації. Право власності на дані, контроль над витратами, відповідність нормативним вимогам — причини зрозумілі.

Але ринок самостійно розгорнутої документації перенасичений. Wiki-платформи, генератори статичних сайтів, бази знань і гібридні інструменти — кожен із власними компромісами. Цей посібник допоможе розібратися у різноманітті варіантів.

## Що робить платформу документації якісною?

1. **Складність встановлення.** Запускається менш ніж за 5 хвилин?
2. **Досвід роботи в редакторі.** Вебредактор? Markdown-орієнтований, WYSIWYG чи обидва варіанти?
3. **Інтеграція з git.** Справжня двостороння синхронізація чи лише косметичний односпрямований експорт?
4. **Публікація результату.** Професійний публічний сайт документації?
5. **Якість пошуку.** Повнотекстовий пошук із підсвіченням?
6. **Навантаження на обслуговування.** Резервне копіювання бази даних, оновлення, патчі безпеки?
7. **Портативність даних.** Чистий експорт у markdown?

## Претенденти

### Wiki.js
Wiki на базі Node.js. Кілька сховищ для зберігання даних. Потребує бази даних. Інтеграція з git — одностороння. Найкраще підходить для традиційної wiki без справжнього git.

### BookStack
Wiki на базі PHP. Просте встановлення. Немає інтеграції з git. Контент заблокований у MySQL. Найкраще підходить для нетехнічних команд.

### Outline
Сучасна база знань, блоковий редактор у стилі Notion. Красивий інтерфейс. Немає інтеграції з git. Складний хостинг (PostgreSQL, Redis, S3). Найкраще підходить для внутрішньої бази знань.

### Docusaurus (Meta)
Генератор статичних сайтів на базі React. Відмінний результат публікації. Немає вебредактора. Суто git-процес. Найкраще підходить для команд розробників.

### MkDocs / Material for MkDocs
Генератор статичних сайтів на базі Python. Чиста тема Material. Немає вебредактора. Найкраще підходить для Python-команд.

### Valoryx
Платформа на базі Go. Один бінарний файл, нульові залежності, встановлення за 30 секунд. Справжня двостороння синхронізація з git (Content Ledger). WYSIWYG-редактор. Публікація документації. Пошук FTS5. Безкоштовно для 5 редакторів.
